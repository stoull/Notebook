---

### 一、系统整体负载（头部信息）

- **`top - 13:47:31 up 23:50, 2 users`**  
  当前时间下午 1:47，系统已连续运行 **23小时50分钟**，有2个用户登录（可能是SSH和本地桌面）。

- **`load average: 0.55, 0.55, 0.62`**  
  这是**1分钟、5分钟、15分钟**的平均负载（指等待CPU处理的进程数 + 不可中断的进程数）。  
  树莓派4B是**4核CPU**，负载长期在 **0.6** 左右，意味着平均每个核心只有 0.15 个任务在排队。**远低于临界值4.0**，系统非常轻松。

---

### 二、CPU 和内存使用率（核心指标）

**CPU 行：**
```
%Cpu(s):  6.8 us,  7.2 sy,  0.0 ni, 86.0 id,  0.0 wa,  0.0 hi,  0.0 si,  0.0 st
```

- **总使用率** = `us(6.8%) + sy(7.2%) + ni(0%)` = **14.0%**
- **空闲率 86.0%** 说明 CPU 大部分时间在休息。
- **`us` 6.8%**（用户态）：运行应用程序（如推流软件、桌面环境）的开销。
- **`sy` 7.2%**（内核态）：系统调用开销，比如网络发送数据包、读写摄像头设备——这在推流场景中很正常。
- **`wa` 0.0%**（I/O等待）：没有因为等待硬盘或USB数据而卡顿，说明存储和摄像头读取很顺畅。
- **`hi/si` 均为0**：没有硬件或软件中断干扰，系统很干净。

**内存行：**
```
MiB Mem :   3744.4 total,   2058.5 free,    438.2 used,   1247.7 buff/cache
MiB Swap:    100.0 total,    100.0 free,      0.0 used.   3227.8 avail Mem
```

- **总内存 3.7GB**，实际使用仅 **438MB**（这还不包括缓存）。
- **`buff/cache` 占 1.2GB**：这是Linux把空闲内存拿来作**磁盘/文件缓存**，用于加速读写。如果应用程序需要内存，这部分会立即释放，所以不用紧张。
- **可用内存 `avail Mem` 高达 3.2GB**，这是系统估算的**真正可分配给新程序的内存**，非常充裕。
- Swap（交换分区）完全没有使用，说明物理内存绰绰有余。

# Linux df（Disk Free）,du（Disk Usage）查看磁盘空闲和用量

记Linux命令最怕死记硬背，尤其`du`和`df`这俩“双胞胎”——长得像，但用途完全不同。**别背字母，背它们背后的“故事”和“场景”**，一旦有了画面感，一辈子都忘不掉。

给你一套**“三脑记忆法”**（画面感 + 英文拆解 + 生活类比），今天试完明天想忘都难：

---

### 1. 第一招：看字母拆解（英文原罪法）

- **`df` = Disk Free**（磁盘空闲）
  - **记住**：看到 `f` 就想 **Free（空闲/剩余）**。它永远在问你：“**我还有多少剩余空间？**”
  - **场景**：当你**还没创建文件**，只是想看看硬盘还剩多少G，用 `df -h`。

- **`du` = Disk Usage**（磁盘使用量）
  - **记住**：看到 `u` 就想 **Usage（使用/占用量）**。它永远在问你：“**这个文件夹到底有多大？**”
  - **场景**：当你**已经创建了文件**，想知道某个电影文件夹占了多大地方，用 `du -sh 文件夹名`。

---

### 2. 第二招：画面感记忆（买菜VS查账）

把Linux磁盘想象成一个**大仓库（硬盘）**：

- **`df` 是“仓库管理员”**：
  他站在门口看**全局**，只看整块硬盘分了几区（/dev/sda1），每区总大小多少，还剩多少空位。**他不关心你具体那堆货有多大，只看剩余空位。**

- **`du` 是“仓库盘点员”**：
  他拿着尺子钻进去，**实际去量**你指定的那堆货（文件夹）的长宽高。**他不关心整个仓库多大，只关心你问的那堆东西实际占地多少。**

**一句话总结画面**：
**`df`看“全局剩余”，`du`看“局部实占”。**

---

### 3. 第三招：死穴大法（记他们的“致命区别”）

这是最容易混淆的点，记牢这个**“删文件翻车现场”**，你就彻底通了：

> **情景**：你删了一个正在被进程打开的大日志文件（比如 `rm -f access.log`）。
> 1. 你用 **`df`** 看剩余空间——**没变！还是满的！**（因为你删了文件，但进程没释放，空间没还给系统）。
> 2. 你用 **`du`** 看当前目录总大小——**变小了！**（因为du不查进程，只扫目录树，发现目录里没这个文件了）。

**记忆锚点**：
- **`df`** 查的是 **“文件系统元数据”**（系统层面的记账本），哪怕文件删了，只要进程占用，记账本就不释放。
- **`du`** 查的是 **“目录树实体”**（实实在在去文件夹里加总），目录里没有就是没有。

**所以，当发现`df`和`du`结果对不上时，说明有文件被删但进程没释放！** 这是运维必考知识点，记住这个场景，这两个命令你就刻在骨子里了。

---

### 4. 组合拳实战（形成肌肉记忆）

不要单独记，要记**黄金搭档组合**：

- **看硬盘还剩多少**：`df -h` （`h`=human，人性化显示GB/MB）
- **看当前文件夹多大**：`du -sh *` （`s`=summary汇总，`h`=人性化，`*`=所有子文件夹）

**背顺口溜**：
> **“查全盘用 df（Free），查大小用 du（Usage）；**
> **df 看系统，du 看文件；**
> **对不上号找 lsof（找那个没释放的进程）！”**

---

### 5. 终极偷懒法（如果还是记混）

把你服务器上的 `~/.bashrc` 里加上这两行别名：

```bash
alias 还剩多少='df -h'
alias 这多大='du -sh'
```

以后忘记英文，直接敲中文拼音，用一次记一次，慢慢就条件反射了。
