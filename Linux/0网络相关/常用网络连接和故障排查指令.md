# 常用网络连接和故障排查指令

### SSH连接问题排查示例

* 1. 先用nslookup检查域名解析是否正常: `nslookup xxx.com`

```
nslookup xxx.com
作用：查询域名解析信息
功能：查看"birthday.com"域名对应的IP地址
用途：确认域名是否正确解析到目标服务器IP，排查DNS问题
```

* 2. 再用nc测试端口是否开放: `nc -vz xxx.com 31800`

```
nc -vz xxx.com 31800
作用：测试指定端口的网络连通性
参数解释：

-v：详细输出
-z：只扫描端口，不发送数据
功能：测试"birthday.com"的31800端口是否开放并可连接
用途：快速检查目标端口是否可达，不涉及协议层面的握手
```

 * 3. 最后用ssh -vvv进行详细连接测试，查看具体错误信息: `ssh -vvv -p 31800 hut@xxx.com`

```
ssh -vvv -p 31800 hut@xxx.com

作用：使用SSH协议连接到远程服务器
参数解释：

-vvv：启用最详细的调试模式（3个v表示最高级别的详细输出）
-p 31800：指定使用31800端口（SSH默认是22端口）
johh@birthday.com：以用户"johh"的身份连接到"birthday.com"主机
用途：详细显示SSH连接的每一个步骤，常用于排查连接问题
```


### AI ####

这里总结一些常用的网络连接和故障排查指令，按类别组织：

## 🔍 **基础连通性测试**

### **ping**
```bash
ping example.com
ping 8.8.8.8
```
- 测试网络连通性（ICMP协议）
- 参数：`-c 4`（发送4个包）、`-i 0.5`（间隔0.5秒）

### **traceroute** / **tracepath** / **mtr**
```bash
traceroute google.com
mtr google.com  # 实时追踪，更详细
```
- 显示数据包到达目标的路径和每一跳的延迟

## 🌐 **DNS查询**

### **dig**（推荐）
```bash
dig example.com
dig example.com MX    # 查询邮件记录
dig @8.8.8.8 example.com  # 指定DNS服务器
```
- 功能强大的DNS查询工具，输出详细

### **nslookup**
```bash
nslookup example.com
nslookup -type=MX example.com
```
- 交互式DNS查询，Windows/Linux通用

### **host**
```bash
host example.com
host 8.8.8.8         # 反向DNS查询
```
- 简单的DNS查询工具

## 🔌 **端口和连接测试**

### **netstat** / **ss**（现代替代）
```bash
ss -tuln             # 查看本地监听端口
ss -tunp             # 查看所有连接和进程
netstat -tuln        # 传统方式
```
- 查看网络连接、路由表、接口统计

### **nc** (netcat) - "网络瑞士军刀"
```bash
nc -vz example.com 80    # 测试TCP端口
nc -vzu example.com 53   # 测试UDP端口
nc -l 8080               # 监听端口
```
- 端口扫描、文件传输、端口监听

### **telnet**
```bash
telnet example.com 80
```
- 测试TCP端口连通性（也可用于HTTP等协议测试）

## 📡 **网络配置查看**

### **ifconfig** / **ip**（推荐）
```bash
ip addr show           # 查看IP地址
ip route show          # 查看路由表
ifconfig -a            # 传统方式
```

### **route**
```bash
route -n              # 查看路由表
```

## 🌊 **网络抓包分析**

### **tcpdump**
```bash
tcpdump -i eth0 port 80
tcpdump -i any host 8.8.8.8
tcpdump -w capture.pcap  # 保存到文件
```
- 命令行抓包工具

### **wireshark** / **tshark**
```bash
tshark -i eth0 -f "port 80"
```
- 图形化/命令行抓包分析

## 🔗 **HTTP/Web相关**

### **curl**
```bash
curl -I https://example.com    # 只显示头部
curl -v https://example.com    # 详细输出
curl -o file.html https://...  # 保存到文件
```
- 传输数据，支持多种协议

### **wget**
```bash
wget https://example.com/file.zip
wget --spider https://example.com  # 只检查是否存在
```
- 下载文件

## 📊 **网络统计**

### **nethogs**
```bash
nethogs eth0
```
- 按进程显示网络带宽使用情况

### **iftop**
```bash
iftop -i eth0
```
- 实时显示网络带宽使用

### **bmon**
```bash
bmon
```
- 监控网络带宽和错误统计

## 🛠️ **高级工具**

### **nmap**
```bash
nmap -sS example.com          # SYN扫描
nmap -p 22,80,443 example.com # 扫描指定端口
nmap -O example.com           # 操作系统探测
```
- 网络发现和安全扫描

### **iperf** / **iperf3**
```bash
# 服务器端
iperf3 -s
# 客户端
iperf3 -c server_ip
```
- 网络带宽测试

### **ssh**（调试模式）
```bash
ssh -vvv user@host            # 最高级别调试
ssh -G user@host              # 查看配置
```

## 🎯 **实用排查流程**

1. **快速连通性检查**：
```bash
ping target.com
nslookup target.com
traceroute target.com
```

2. **端口/服务检查**：
```bash
nc -vz target.com 端口号
telnet target.com 端口号
curl -I target.com
```

3. **本地网络检查**：
```bash
ip addr show
ip route show
ss -tuln
```

4. **高级分析**：
```bash
tcpdump -i any host target.com
mtr target.com
```

## 💡 **常用组合命令**

```bash
# 检查DNS解析和端口开放
dig +short example.com | xargs -I {} nc -zv {} 443

# 批量检查多个端口
for port in 80 443 22; do nc -zv example.com $port; done

# 持续ping监控
ping -i 1 example.com | while read pong; do echo "$(date): $pong"; done
```

这些工具涵盖了从基础连通性测试到高级网络分析的大部分场景，掌握它们能有效解决90%以上的网络问题。
