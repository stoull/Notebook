#【26OA-22】编辑电站页面增加第三方授权信息 需求描述

### 1
在newStationSettingVC.m文件中，在self.twoView和threeView之间，即self.twoView的下方，threeView的上方增加 第三方授权 的相关功能的界面。界面如‘第三方授权-未授权.png’ 和 ‘第三方授权-已授权.png’所示，是同一个界面的两种状态。
当第三方授权-未授权的时候，如‘第三方授权-未授权.png’所展示，40x40pt的图标，逆变器型号，SN，授权码默认值为-，剩余有效期默认值为‘-天’，右上角有灰色的‘未授权’状态显示。
当第三方授权功能整体的界面如’第三方授权-已授权.png‘图所展示，逆变器型号，SN，授权码，剩余有效期，都有值，并且右上角有绿色的‘已授权’状态显示。
接口信息在后面

交互逻辑为： 

1.当进入这个界面的时候，调用下面的 ‘获取电站下逆变器授权码接口’，如果成功获取到授权信息，则显示 ‘第三方授权-已授权.png’， 否则显示‘第三方授权-未授权.png’
2. 如果没有获取到授权信息则显示‘第三方授权-未授权.png’
3. 点击 获取授权码按钮 则调用 ‘生成电站下逆变器授权码接口’，如果生成成功则刷新界面，获取授权码按钮 用户可以随时点击
4. 授权码 后面有一个复制按钮，用户点击后复制授权码到剪切板，并提示‘已复制’

注意：老代码非必要不更改，担心影响功能，除非一定要改


网络使用类似方法中的-(void)requestData方法，使用[BaseRequest requestWithMethodResponseJsonByGet:HEAD_URL paramars:@{@"plantId":plantId} paramarsSite:@"/newPlantAPI.do?op=getPlant" sucessBlock:^(id content)调用。下面的http://xx.com都使用HEAD_URL表示即可。


获取电站下逆变器授权码接口：
post http://xx.com/newTwoPlantAPI.do?op=getInverterAuthCodes
传参数名 plantId string	电站ID
返回示例
失败：{
    "result": 0,//获取失败
    "msg": "",
    "obj": null
}
成功：{
    "result": 1,
    "msg": "Success",
    "obj": [
        {
            "authCode": "0000001CVGPSW2CA",//授权码
            "expireTime": "0",//授权码有效期(填)
            "deviceModel": "WIS 215K-AM",//逆变器型号
            "deviceSn": "LCQ1F7H00E00000000000000000000"//设备SN
        }
    ]
}

生成电站下逆变器授权码接口：
post http://xx.com/newTwoPlantAPI.do?op=generateInverterAuthCodes
传参数名 plantId string	电站ID
返回示例
{
    "result": 0,//获取失败
    "msg": "",
    "obj": null
}
{
    "result": 1,
    "msg": "Success",
    "obj": [
        {
            "authCode": "0000001CVGPSW2CA",//授权码
            "expireTime": "0",//授权码有效期
            "deviceModel": "WIS 215K-AM",//逆变器型号
            "deviceSn": "LCQ1F7H00E00000000000000000000"//设备SN
        }
    ]
}
